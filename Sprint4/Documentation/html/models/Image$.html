  32. def insert(filename: String, deploymentId: Long, json: String, addedBy: String, originalFilename: String): <error>

    Insert a new image.

    Insert a new image. insert if an image with the same path and deployment does not exist

    images with duplicate paths may lead to various issues due to the fact that the AWS can have only one object with the given key (i.e., filename) and adding new objects with the same key will overwrite the existing object

    duplicate paths may result form the fact that some deployments have images that were taken within the same second; due to our file naming (based on the timestamp up to a second) this will result and multiple db records pointing to the same AWS file; this in turn would become a problem if one of the images needs to be deleted from both db and AWS

    having added a unique constraint on the filename column, it is still preferable to have a WHERE NOT EXISTS clause for performance reasons: appears slightly faster than entering an exception block fo each duplicate image; however, just having WHERE NOT EXISTS without the unique constraint is not sufficient as it is subject to a race condition (that was frequently occurring during batch uploads)
